# 安全策略

## 🔒 报告安全漏洞

如果您发现安全漏洞，请通过 GitHub Issues 私下报告，不要在公开渠道讨论。

## ⚠️ 安全注意事项

### API 密钥保护

- 所有 API 密钥必须通过环境变量配置，**禁止**提交到代码仓库
- `.env` 文件已加入 `.gitignore`
- 使用 `.env.example` 作为配置模板
- 生产环境务必修改默认的 `API_SECRET`

### 依赖安全

- 定期运行 `npm audit` 检查漏洞
- 及时更新依赖版本
- 使用 `npm audit fix` 自动修复

### 部署安全

#### 生产环境建议

1. **使用 HTTPS**
   - 配置 SSL/TLS 证书
   - 启用 HSTS

2. **API 认证**
   - 设置强 `API_SECRET`
   - 启用速率限制
   - 配置 `CORS_ORIGIN`

3. **容器安全**
   - 以非 root 用户运行容器
   - 使用只读文件系统
   - 限制容器能力

4. **网络隔离**
   - 使用防火墙规则
   - 限制数据库访问
   - 启用 VPC（云部署）

### 输入验证

- 所有用户输入必须经过验证
- 使用 Zod 进行 schema 验证
- 防止 SQL 注入（已使用参数化查询）
- 防止 XSS 攻击

### 日志与监控

- 记录安全相关事件
- 设置异常告警
- 定期审查日志
- 避免记录敏感信息

## 🆘 安全事件响应

1. **立即隔离**受影响的系统
2. **评估影响范围**
3. **修复漏洞**
4. **通知相关方**
5. **复盘改进**

## 📋 合规提示

- 遵守目标地区的数据保护法规
- 实施数据加密
- 配置数据保留策略
- 定期安全审计
